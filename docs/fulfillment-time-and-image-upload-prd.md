# PRD: 수령 가능 시간 자동 선택 및 상품 이미지 파일 업로드

## 1. 배경

고객 주문 화면에서 수령 날짜와 픽업 시간이 비어 있으면 고객이 언제부터 주문 가능한지 직접 판단해야 한다. 꽃 제작에는 준비 시간이 필요하므로 현재 시각 기준 최소 3시간 이후만 선택 가능해야 한다.

관리자 상품관리에서는 이미지 경로 텍스트 입력만 가능해 실제 운영자가 로컬에 있는 이미지 파일을 선택해 상품 대표 이미지로 반영하기 어렵다.

## 2. 목표

- 고객 주문 화면에서 가장 빠른 수령 가능 날짜와 시간이 자동 선택되게 한다.
- 수령 가능 최소 시점은 현재 시각 기준 3시간 이후로 계산한다.
- 3시간 이후 시점이 18:00을 넘으면 다음날 09:00을 기본 수령 시점으로 한다.
- 픽업 시간은 09:00~18:00 사이 30분 단위만 선택 가능하게 유지한다.
- 관리자 상품관리에서 로컬 이미지 파일을 선택하면 상품 대표 이미지로 반영한다.

## 3. 범위

### 포함

- 수령 가능 최소 날짜/시간 계산
- 수령 날짜 기본값 자동 설정
- 픽업 시간 기본값 자동 설정
- 선택 날짜가 최소 날짜일 때 불가능한 픽업 시간 숨김
- 배송 시간대도 최소 수령 가능 시점 기준으로 선택지 제한
- 관리자 상품 이미지 파일 선택
- 선택한 이미지를 브라우저 저장소에 data URL로 저장
- 상품관리와 고객 주문 화면에 업로드 이미지 즉시 반영

### 제외

- 서버 파일 업로드
- 클라우드 스토리지 연동
- 이미지 압축/리사이징
- 업로드 파일 삭제 관리

## 4. 정책

### 수령 가능 시점

- 기준: 현재 시각 + 3시간
- 영업시간: 09:00~18:00
- 분 단위: 00분, 30분
- 현재 시각 + 3시간이 09:00 이전이면 당일 09:00부터 가능
- 현재 시각 + 3시간이 18:00을 넘으면 다음날 09:00부터 가능
- 현재 시각 + 3시간이 18:00이면 당일 18:00 선택 가능

### 이미지 업로드

정적 MVP에서는 브라우저 보안상 로컬 파일의 실제 절대 경로를 그대로 사용할 수 없다. 따라서 파일 선택 시 브라우저가 읽은 이미지 데이터를 data URL로 변환해 `localStorage`에 저장한다. 이는 MVP에서 "업로드된 것처럼" 동작하는 방식이며, 실제 운영 버전에서는 서버 또는 스토리지 업로드로 교체해야 한다.

- 고객 주문 화면에도 변경 이미지가 반영됨
