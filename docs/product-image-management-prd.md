# PRD: 상품관리 기본 이미지 표시 기능

## 1. 배경

현재 상품관리 화면은 상품명, 설명, 가격, 판매 가능 여부만 표시한다. 꽃 판매 서비스에서는 상품 이미지가 주문 전환과 관리자 운영에 중요하므로, 상품관리 화면에서 상품별 대표 이미지를 바로 확인하고 수정할 수 있어야 한다.

## 2. 목표

- 상품관리 화면에서 상품별 대표 이미지를 표시한다.
- 고객 주문 화면의 상품 카테고리 카드에도 대표 이미지를 표시한다.
- 기본 상품에는 실제 사진 느낌의 추천 기본 이미지를 제공한다.
- 외부 네트워크에 의존하지 않도록 기본 이미지는 저장소 내부 에셋으로 관리한다.
- 일러스트 이미지는 fallback으로 보관하고, 기본 노출 이미지는 사진 에셋을 사용한다.

## 3. 범위

### 포함

- 상품별 기본 대표 이미지 에셋 추가
- 상품 데이터에 `imageUrl` 필드 추가
- 고객 주문 화면 카테고리 카드에 이미지 표시
- 관리자 상품관리 목록에 이미지 미리보기 표시
- 관리자 권한에서 이미지 경로 수정 가능
- 직원 권한에서는 이미지 경로 수정 불가
- README 문서 업데이트

### 제외

- 실제 이미지 업로드
- 이미지 리사이징/압축
- 클라우드 스토리지 연동
- 드래그 앤 드롭 이미지 관리
- 상품별 다중 갤러리 편집 UI

## 4. 기본 이미지 정책

기본 노출 이미지는 다음 상품 카테고리에 제공한다.

- 꽃다발: `assets/bouquet-photo.jpg`
- 꽃바구니: `assets/basket-photo.jpg`
- 화분: `assets/plant-photo.jpg`
- 맞춤주문: `assets/custom-photo.jpg`

SVG 이미지는 fallback 또는 네트워크 없이 빠르게 확인하기 위한 보조 에셋으로 유지한다.

기본 사진은 운영자가 실제 제작 사진을 준비하기 전까지 보여주는 추천 이미지로 사용한다. 실제 운영 전에는 매장에서 직접 촬영한 제작 예시 사진으로 교체하는 것을 권장한다.

## 5. 기능 요구사항

### 5.1 고객 주문 화면

- 상품 카테고리 카드 상단에 대표 이미지를 표시한다.
- 이미지가 없거나 로드 실패 시 기본 배경색 영역으로 대체한다.
- 상품명, 설명, 선택 상태가 이미지와 겹치지 않아야 한다.

### 5.2 관리자 상품관리 화면

- 상품 목록에 대표 이미지 썸네일을 표시한다.
- 관리자 권한은 이미지 경로를 수정할 수 있다.
- 직원 권한은 이미지 경로를 볼 수 있지만 수정할 수 없다.
- 이미지 경로 변경 후 고객 주문 화면에도 변경된 이미지가 반영되어야 한다.
