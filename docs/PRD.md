# PRD: 온라인 꽃 주문 및 관리자 운영 시스템 MVP

## 1. 제품 개요

### 제품명

온라인 꽃 예약 주문 및 주문관리 시스템

### 목적

오프라인 꽃집의 주문 접수 채널을 온라인으로 확장한다. 고객은 웹사이트에서 꽃 상품을 예약 주문하고, 관리자는 접수된 주문을 확인해 제작 가능 여부와 수령 일정을 확정한다.

### 핵심 방향

MVP는 완전한 쇼핑몰이 아니라 **예약 주문 접수 + 관리자 확인 기반 주문관리 시스템**이다.

온라인 결제, 회원가입, 마이페이지, 재주문 기능은 2차 범위로 분리한다.

## 2. 문제 정의

현재 오프라인 꽃 판매는 전화, 방문, 카카오톡 상담 중심으로 운영된다. 이 방식은 다음 문제가 있다.

- 주문 정보가 흩어져 관리됨
- 수령일, 시간, 색감, 예산 등 핵심 정보 누락 가능
- 주문 상태 확인이 어렵고 누락 위험이 있음
- 고객 이력과 선호도를 체계적으로 관리하기 어려움
- 온라인 유입 고객을 주문으로 전환하기 어려움

## 3. 목표

### MVP 목표

- 고객이 온라인에서 비회원으로 꽃 주문을 접수할 수 있다.
- 관리자가 주문을 확인하고 상태를 관리할 수 있다.
- 상품, 옵션, 판매 가능 여부를 관리할 수 있다.
- 고객 이력과 메모를 이름 + 전화번호 기준으로 관리할 수 있다.
- 신규 주문 발생 시 고객과 관리자에게 문자 알림을 보낼 수 있다.
- 1~2주 테스트 운영 후 정식 오픈할 수 있다.

### 성공 기준

- 고객이 별도 설명 없이 주문서를 완료할 수 있음
- 관리자가 오늘/내일 제작할 주문을 빠르게 확인할 수 있음
- 주문 누락 없이 접수/확정/완료 상태를 관리할 수 있음
- 오프라인 상담보다 주문 정보 누락이 줄어듦

## 4. 사용자

### 고객

- 꽃다발, 꽃바구니, 화분, 맞춤주문을 예약하려는 일반 고객
- 회원가입 없이 빠르게 주문하고 싶어 함
- 주문 후 관리자 확인을 기다림

### 관리자

- 주문 전체 관리
- 상품, 가격, 옵션, 이미지 관리
- 고객 이력 및 메모 관리
- 기본 통계 확인

### 직원

- 주문 확인 및 상태 변경
- 상품 판매 가능 여부만 변경

## 5. MVP 범위

### 포함

- 비회원 주문 접수
- 고객 주문 페이지
- 관리자 주문관리
- 상품관리
- 고객관리
- 관리자 대시보드 기본 통계
- 고객 문자 접수 알림
- 관리자 화면 알림 및 문자 알림
- 배송 가능 지역/배송비 관리
- 1~2주 테스트 운영

### 제외

- 온라인 결제
- 소셜 로그인
- 마이페이지
- 주문내역 조회
- 재주문
- 리뷰
- 쿠폰/포인트
- 재고관리
- 배송 배차
- 화환 주문
- 정기구독

## 6. 고객 주문 플로우

1. 상품 카테고리 선택
2. 예산 선택
3. 색감 선택
4. 분위기 선택
5. 용도 선택
6. 메시지 카드 문구 입력
7. 수령 방식 선택
8. 날짜/시간 선택
9. 고객 정보 입력
10. 주문 접수 완료

### 상품 카테고리

- 꽃다발
- 꽃바구니
- 화분
- 맞춤주문

화환은 MVP에서 제외한다.

### 예산

정해진 가격대 중 선택한다. 상품별 최소 금액을 설정할 수 있어야 한다.

예시:

- 꽃다발: 3만원부터
- 꽃바구니: 5만원부터
- 화분: 5만원부터
- 맞춤주문: 상담 필요

### 색감

예시:

- 핑크
- 화이트
- 옐로우
- 레드
- 퍼플
- 혼합
- 기타

### 분위기

예시:

- 화사한
- 차분한
- 고급스러운
- 귀여운
- 풍성한
- 기타

### 용도

- 생일
- 기념일
- 프로포즈
- 졸업
- 입학
- 부모님 선물
- 기타

### 메시지 카드

선택 입력으로 제공한다.

### 참고 사진 업로드

맞춤주문에만 제공한다.

## 7. 수령 및 배송 정책

### 수령 방식

- 매장 픽업
- 근거리 직접 배송

### 시간 선택

- 픽업: 정확한 시간 선택
- 배송: 시간대 선택

배송 시간대 예시:

- 10:00~12:00
- 12:00~14:00
- 14:00~16:00
- 16:00~18:00

### 배송 지역

- 구/동 단위로 배송 가능 지역 관리
- 지역별 배송비 설정
- 배송 불가 지역은 선택 불가 또는 상담 안내

## 8. 주문 상태

주문 상태는 단순하게 관리한다.

- 접수
- 확인중
- 확정
- 완료
- 취소

주문은 자동 확정되지 않는다. 관리자가 재료, 제작 가능 시간, 배송 가능 여부, 요청사항을 확인한 뒤 직접 확정한다.

결제는 MVP에서 온라인 연동하지 않는다. 계좌이체 또는 현장결제로 처리하며, 입금 여부는 체크박스 또는 메모로 관리한다.

## 9. 관리자 기능

### 대시보드

관리자 첫 화면에 다음 정보를 표시한다.

- 오늘 수령 예정 주문
- 내일 수령 예정 주문
- 신규 접수 주문
- 이번 달 완료 주문 금액
- 취소 주문 수

### 주문관리

주문 목록에서 확인할 항목:

- 주문번호
- 상태
- 주문자 이름
- 전화번호
- 상품 카테고리
- 예산
- 수령 방식
- 수령 날짜
- 수령 시간 또는 시간대
- 접수일
- 입금 확인 여부

필터:

- 상태
- 날짜
- 픽업/배송

주문 상세:

- 고객 정보
- 주문 옵션
- 요청사항
- 메시지 카드
- 배송 정보
- 참고 사진
- 주문별 메모
- 고객별 메모
- 상태 변경
- 입금 확인

### 주문 캘린더

- 수령일 기준으로 주문 표시
- 오늘/내일 주문 확인
- 시간대별 주문 보기

## 10. 상품관리

관리자는 다음 항목을 관리할 수 있다.

- 상품 카테고리명
- 설명
- 기본 이미지
- 실제 제작 예시 사진
- 색감/분위기별 갤러리
- 가격대 옵션
- 상품별 최소 금액
- 픽업 가능 여부
- 배송 가능 여부
- 판매 가능 여부
- 노출 여부

직원은 **판매 가능 여부만 변경**할 수 있다.

## 11. 고객관리

고객은 비회원 기준으로 관리한다.

식별 기준:

- 이름
- 전화번호

고객 상세에서 확인할 항목:

- 이름
- 전화번호
- 총 주문 수
- 최근 주문일
- 주문 이력
- 고객별 메모
- 고객 태그

태그 예시:

- 단골
- 기업
- VIP
- 주의
- 선호 핑크톤
- 선호 화이트톤

## 12. 알림

### 고객 알림

주문 접수 시 문자 발송.

내용:

- 주문 접수 완료
- 관리자 확인 후 확정 예정
- 카카오톡 상담 링크

### 관리자 알림

신규 주문 접수 시:

- 관리자 화면에 신규 주문 표시
- 관리자에게 문자 알림 발송

- 고객관리

- 주문내역 확인
- 비슷하게 재주문
- 온라인 카드/간편결제
- 취소 요청
- 주소록 저장
