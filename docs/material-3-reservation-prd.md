# PRD: Material 3 디자인 개선 및 예약 시간 선택 UX

## 1. 배경

현재 MVP는 기능 검증용 정적 웹앱으로 구현되어 있으나, 고객 주문 화면과 관리자 화면의 시각적 완성도가 낮고 일부 환경에서 디자인이 적용되지 않은 것처럼 보일 수 있다. 또한 예약 시간 입력이 브라우저 기본 `time` 입력에 의존해 사용자가 허용되지 않은 시간을 직접 입력할 여지가 있다.

## 2. 목표

- 고객 주문 화면과 관리자 화면에 Google Material Design 3 스타일을 적용한다.
- 예약 시간 선택은 고객이 선택 가능한 값만 고를 수 있도록 제한한다.
- 픽업 시간은 09:00부터 18:00까지 30분 단위로만 선택 가능하게 한다.
- 배송 시간대도 영업시간인 09:00부터 18:00 사이에서만 선택 가능하게 한다.
- 기존 MVP 기능은 유지한다.

## 3. 범위

### 포함

- Material 3 기반 디자인 토큰 적용
- 버튼, 카드, 입력 필드, 선택 필드, 탭, 배지 스타일 개선
- 고객 주문 화면 레이아웃 개선
- 관리자 화면 레이아웃 개선
- 픽업 시간 선택 UI를 직접 입력 방식에서 선택식으로 변경
- 배송 시간대 옵션을 09:00~18:00 범위로 정리
- README 실행 안내 보강

### 제외

- Material Web Components 패키지 설치
- 빌드 도구 도입
- 백엔드 연동
- 실제 SMS 발송
- 실제 운영 배포

## 4. 사용자 요구사항

### 고객

- 주문 화면이 전문적인 서비스처럼 보여야 한다.
- 예약 가능한 시간만 선택할 수 있어야 한다.
- 09:00~18:00 외 시간은 고를 수 없어야 한다.
- 분 단위는 00분 또는 30분만 선택 가능해야 한다.

### 관리자

- 관리자 화면에서 주문, 상품, 고객, 알림 정보를 명확하게 구분해서 볼 수 있어야 한다.
- 상태, 권한, 판매 가능 여부 등 운영 정보가 시각적으로 잘 드러나야 한다.

## 5. 기능 요구사항

### 5.1 Material 3 스타일

- `--md-sys-color-*` 형태의 디자인 토큰을 CSS 변수로 정의한다.
- 주요 화면 배경은 surface 계열 색상을 사용한다.
- 주요 액션 버튼은 filled button 형태로 표현한다.
- 보조 액션 버튼은 tonal 또는 outlined button 형태로 표현한다.
- 카드와 패널은 Material 3 card 스타일에 맞춰 낮은 elevation과 명확한 surface 구분을 제공한다.
- 입력 필드는 filled/outlined text field에 가까운 형태로 통일한다.
- 모바일 화면에서도 입력 요소와 버튼이 깨지지 않아야 한다.

### 5.2 예약 시간 선택

- 픽업 시간은 `<select>`로 제공한다.
- 선택지는 다음 규칙으로 생성한다.
  - 시작: 09:00
  - 종료: 18:00
  - 간격: 30분
- 허용 예시:
  - 09:00
  - 09:30
  - 10:00
  - 10:30
  - ...
  - 17:30
  - 18:00
- 사용자가 직접 임의 시간을 입력할 수 없어야 한다.

### 5.3 배송 시간대

- 배송 시간대는 09:00~18:00 범위 안에서 선택하게 한다.
- 기본 선택지는 다음으로 한다.
  - 09:00~11:00
  - 11:00~13:00
  - 13:00~15:00
  - 15:00~17:00
  - 17:00~18:00
