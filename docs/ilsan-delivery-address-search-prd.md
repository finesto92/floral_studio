# PRD: 일산 한정 배송 및 주소 검색 UI

## 1. 배경

현재 MVP는 강남구, 서초구, 송파구 같은 예시 배송 지역을 사용하고 있다. 실제 운영 정책은 일산 지역 한정 배송이며, 배송비는 고정 무료/정액이 아니라 퀵배송비를 별도로 청구하는 방식이다.

고객이 배송 주소를 입력할 때는 참고 페이지처럼 우편번호/주소 검색 UI를 제공해 도로명 또는 동/읍/면/리+번지 기준으로 주소를 찾고 선택할 수 있어야 한다.

참고 UI:

- 산림청 우편번호검색: https://www.forest.go.kr/kfsweb/kfi/kfs/newZipcode/newZipcodeView2.do

## 2. 목표

- 배송 가능 지역을 일산으로 제한한다.
- 배송 주문 시 퀵배송비가 별도 청구된다는 점을 명확히 안내한다.
- 주소 입력은 직접 입력만이 아니라 주소 검색 모달을 통해 선택할 수 있게 한다.
- 검색 UI는 도로명 또는 동/읍/면/리+번지 입력 방식의 안내를 제공한다.

## 3. 범위

### 포함

- 배송 가능 지역을 일산동구/일산서구 중심으로 변경
- 배송비 표기를 "퀵배송비 별도"로 변경
- 고객 주문 화면 배송 섹션에 퀵배송비 별도 청구 안내
- 주소 검색 버튼 추가
- 주소 검색 모달 추가
- 도로명/동/번지 키워드 검색
- 검색 결과 선택 시 배송 주소 입력란에 자동 반영
- 관리자 주문 상세에서 퀵배송비 별도 안내가 보이도록 배송비 표시 정책 변경

### 제외

- 행정안전부 도로명주소 API 연동
- 실제 우편번호 DB 전체 탑재
- 퀵배송비 자동 계산
- 외부 주소 검색 팝업 연동

## 4. 배송 정책

- 배송 가능 지역: 일산
- MVP 상세 지역:
  - 고양시 일산동구
  - 고양시 일산서구
- 퀵배송비는 주문 확정 전 관리자 안내 후 별도 청구한다.
- 배송 주소가 일산 외 지역인 경우 온라인 주문에서 배송 선택을 제한하거나 상담 안내한다.

## 5. 주소 검색 정책

MVP에서는 샘플 주소 데이터를 사용한다.

검색 입력 안내:

- 도로명(~로, ~길) + 건물번호
- 동/읍/면/리 + 번지

예시:

- 중앙로 1275
- 정발산로 24
- 장항동 868
- 대화동 2600

검색 결과 선택 시 다음 값이 배송 주소에 반영된다.

- 우편번호
- 기본 주소

상세 주소는 고객이 직접 입력한다.
